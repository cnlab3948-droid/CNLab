1. 홈페이지 작동 구조 (전체 요약)

CNLab 홈페이지는 관리의 유연성을 위해 두 가지 방식이 혼합되어 있습니다.

분류 설명 해당 메뉴
1. 자동 연동 (구글 시트) 스프레드시트에 내용만 치면 새로고침 시 1초 만에 홈페이지에 자동 반영되는 영역 공지사항, 실험, 연구, 논문, 학회, Gallery
2. 고정 코딩 (Github 파일) 표시 속도 향상과 섬세한 디자인을 위해 index.html 파일 속 코드를 직접 뜯어 고쳐야 하는 영역 연구진 구성원(Members), 가장 위쪽 대문 글, 맨 아래 연락처/이메일
💡 결론: 자주 바뀌는 새소식들은 구글 시트만 열어서 수정하시면 되고, 구성원 입퇴소 갱신이나 연락처 주소 변경 같은 굵직한 구조 변경은 깃허브에서 index.html 코드를 고치시면 됩니다.

2. 코드를 직접 수정하는 영역 (구성원 등)

구글 시트가 아닌 깃허브 홈페이지(GitHub Repository)에 접속하여 index.html 파일을 클릭하고 우측의 연필 모양(Edit) 버튼을 눌러 코드를 직접 수정해야 하는 영역입니다.

👤 2-1. 구성원(연구원) 추가 및 삭제 방법

index.html 파일의 대략 300번째 줄 즈음을 보면 <!-- M.A. Students --> 와 같이 구성원들이 블록 단위로 묶여져 있는 곳을 발견할 수 있습니다.

[새로운 학생을 추가하고 싶을 때]

기존에 있는 학생의 <div class="researcher-card"> ... </div> 뭉치를 드래그해서 그대로 복사한 뒤, 원하는 위치에 붙여넣고 안에 있는 한글/영어 이름만 갈아끼우세요.

[기존 학생을 석방(졸업/삭제) 시킬 때]

코드 편집기에서 지우고자 하는 학생의 이름(예: 조혜린)을 Ctrl + F 등 검색으로 찾은 뒤, 해당 학생을 감싸고 있는 <div class="researcher-card"> 부터 </div> 까지의 블록을 통째로 지우시면 됩니다.

📲 2-2. 맨 밑 실험실 연락처 및 소셜 아이콘 변경

파일 맨 밑바닥 <footer> 쪽에 위치합니다. 검색기능으로 footer__contact를 찾으시면 바로 나옵니다.

  • 주소를 바꾸려면 <span class="lang-ko">영남대학교 ... </span> 내부 텍스트를 수정하세요.
  • 이메일/번호를 바꾸려면 텍스트뿐만 아니라 <a href="mailto:새메일"> 안에 있는 목적지 주소도 같이 바꿔주셔야 실제로 눌렀을 때 새 주소로 갑니다.

3. 구글 시트 연동 영역 (공지, 사진 등)

이곳은 스프레드시트 하단의 탭 이름 6가지 (공지사항, 설문, 연구, 논문, 학회, gallery) 에 행(Row)을 추가하기만 하면 실시간으로 반영되는 곳입니다.

⚠️ [중요] 필드명 유지 법칙
각 시트의 가장 첫 번째 로우(1행)에 적힌 영어 텍스트(예: title_ko, date)는 홈페이지 회로가 데이터를 읽어내는 열쇠입니다. 절대로 지우거나 오타를 내면 안 됩니다.

공지사항 / 설문 / 연구 탭 작성 예시

title_ko (한글제목), title_en (영문제목), summary (요약) 등을 표의 열에 맞게 채워 넣으시면 그 칸에 해당하는 위치에 글씨가 출력됩니다.

  • 공지사항 탭에서 특정한 글을 맨 위로 올리려면 pinned 열 칸에 true 라고 적으시면 됩니다.
  • 설문 탭에서 status_ko 열을 진행중에서 종료됨으로 바꾸면 홈페이지의 버튼이 회색으로 막히면서 터치가 금지됩니다. (영어는 Closed)

📸 Gallery 탭 작성법

홈페이지 [사진첩] 메뉴에 썸네일을 띄워주는 매우 간단한 탭입니다.

image_url
poster1.jpg
  • 방법 1: image_url 열에 깃허브 스토리지에 올린 파일 이름(예: lab_party.jpg)만 적습니다.
  • 방법 2: 구글 드라이브(또는 다른 웹사이트)의 이미지 직접 보기 링크(https://어쩌구.jpg)를 통째로 적으셔도 인식됩니다.

4. 서버 이미지 폴더 구조 (사진 관리)

홈페이지에 출력될 원본 사진 파일(jpg, png 등)을 업로드 하는 경로는 다음 두 군데입니다.

  • images/members/ 폴더: 교수님 프로필 사진이나, 추후 학생들 개인 증명사진을 띄우고 싶으실 때 넣는 폴더입니다.
  • images/conferences/ 폴더: 주요 사진첩(Gallery)에 나올 사진들을 전부 때려 넣는 장소입니다. 구글 시트 gallery 탭에 파일 이름을 쓸 때 여기에 이미지가 미리 업로드되어 있어야 엑박(깨짐)이 뜨지 않습니다.
업로드 방법: 깃허브 Repository 환경에서 명시된 폴더를 누른 뒤 `Add file` -> `Upload files` 버튼을 눌러 사진을 끌어다 놓으시고 `Commit changes`를 누르면 업로드가 완료됩니다. (파일명에 한글이나 띄어쓰기는 지양하고 summer_conference_2026.jpg 형태를 권장합니다)

5. 구글 시트 탭 추가/연동 가이드 (Apps Script 엔진)

새로운 탭을 만들어서 홈페이지로 날리고 싶거나 앱스 스크립트가 꼬였을 때 사용하는 마법의 코드입니다.
구글 시트 상단 확장 프로그램 -> Apps Script로 들어가서, Code.gs 에 있는 내용을 전부 지우고 아래 코드를 복붙한 뒤 [새 버전으로 배포] 하면 정상화됩니다.

function doGet(e) {
  try {
    var ss = SpreadsheetApp.getActiveSpreadsheet();
    var sheets = ss.getSheets();
    var data = {};
    
    // 존재하는 모든 시트 탭을 순회하며 json 객체로 자동 치환
    for (var i = 0; i < sheets.length; i++) {
        var sheet = sheets[i];
        var sheetName = sheet.getName().toLowerCase(); 
        
        var rows = sheet.getDataRange().getDisplayValues();
        if (rows.length < 2) {
             data[sheetName] = [];
             continue; // 빈 시트 패스
        }
        
        var headers = rows[0];
        var sheetData = [];
        for (var j = 1; j < rows.length; j++) {
             var rowData = {};
             var hasValue = false;
             for (var k = 0; k < headers.length; k++) {
                  // 열의 값이 비어있지 않거나 띄어쓰기만 있는게 아니면
                  if (rows[j][k] && rows[j][k].trim() !== "") {
                       rowData[headers[k].trim()] = rows[j][k].trim();
                       hasValue = true;
                  }
             }
             if (hasValue) sheetData.push(rowData);
        }
        data[sheetName] = sheetData; 
    }
    
    return ContentService.createTextOutput(JSON.stringify({result: "success", data: data}))
      .setMimeType(ContentService.MimeType.JSON);
      
  } catch (err) {
    return ContentService.createTextOutput(JSON.stringify({result: "error", error: err.toString()}))
      .setMimeType(ContentService.MimeType.JSON);
  }
}

6. 체험 모듈 (Playground) 관리 가이드

홈페이지 방문객의 흥미를 유발하기 위해 새로 도입된 체험(BETA) 메뉴의 작동 방식 및 관리 방법입니다.

🧠 1. 3D 뇌 탐색기 (NiiVue)

  • 작동 원리: 의료 영상 뷰어인 NiiVue 라이브러리를 사용하여 브라우저에서 실시간으로 3D 뇌를 렌더링합니다.
  • 사용 모델: models/mni152_2009.mz3 (약 4.7MB의 양반구 전체 뇌 3D 메쉬 파일). 흑백 MRI 단면 대신 실제 뇌와 유사한 살구색 3D 표면으로 출력되도록 설정했습니다.
  • 클릭 인식: MNI 3D 공간 좌표(X, Y, Z)를 계산하여 클릭한 부위가 전두엽, 두정엽, 후두엽, 측두엽, 소뇌 중 어디에 해당하는지 파악 후 정보를 팝업으로 제공합니다.

🧑‍🔬 2. 나와 닮은 뇌 영역 테스트 (Persona Quiz)

  • 유형: 총 5개의 심리/행동 패턴 질문을 통해 4가지 뇌 영역 중 하나를 결과로 도출합니다.
  • 결과 종류:
    1. 전전두피질 (PFC): 이성적, 계획적 리더
    2. 편도체 (AMY): 감수성 풍부, 직관적 공감러
    3. 해마 (HIPPO): 꼼꼼함, 경험 중시 기록가
    4. 후두엽 (OCC): 시각적, 창의적 관찰자

🎯 3. 스트룹 과제 (Stroop Task) 및 랭킹 관리

jsPsych v8 기반으로 제작된 인지 제어 실험입니다. 속도와 정확도, 반응 일관성을 종합하여 100점 만점으로 점수를 매깁니다.

[글로벌 실시간 랭킹 구조]

  • 랭킹 데이터는 구글 스프레드시트에 저장됩니다. (Google Apps Script를 통해 읽기/쓰기)
  • 서버가 응답하지 않을 때를 대비하여 각 PC의 로컬스토리지에도 백업 캐싱됩니다.

[랭킹 초기 설정 방법 (최초 1회)]

  1. CNLab에서 사용 중인 구글 스프레드시트를 엽니다.
  2. 하단에서 "+" 버튼을 눌러 새 시트를 만들고, 이름을 rankings로 설정합니다.
  3. rankings 시트의 1행에 다음 헤더를 입력합니다:
    A1: name   B1: score   C1: accuracy   D1: date
  4. 확장 프로그램 → Apps Script를 열고, 새 스크립트 파일을 만들어 깃허브 저장소의 apps_script_ranking.gs 파일 내용을 전부 복사 → 붙여넣기 합니다.
  5. 배포 → 새 배포 → 유형: 웹 앱 → 액세스: 모든 사용자 → 배포
  6. 생성된 URL을 복사하여 js/experiment.js 파일의 RANKING_GAS_URL 값에 붙여넣습니다.

[랭킹 열람 및 초기화(리셋) 방법]

  • 구글 스프레드시트의 rankings 시트(탭)를 열면 현재까지 등록된 모든 랭킹 데이터를 직접 엑셀 형태로 볼 수 있습니다.
  • 초기화: rankings 시트에서 2행~끝까지 내용을 드래그로 선택 → 삭제하면 랭킹이 즉시 초기화됩니다.
  • 특정 유저 삭제: 해당 유저가 적힌 행만 우클릭 → 행 삭제하면 됩니다.
💡 타임아웃 방지: 랭킹 서버 응답이 5초 이상 지연될 경우, 브라우저가 무한 로딩에 빠지지 않도록 강제로 통신을 끊고 로컬 캐시 기록만 우선적으로 띄워주도록 이중 안전장치가 적용되어 있습니다.
